# capstone_viz

Lean replacement for the 1,718-line `generate_charts_(5).py`. Splits the
work into **prep notebooks** (load CSVs, fit models, save small artifacts)
and **viz notebooks** (read artifacts, render charts). Every logical step
that V5 did inline is now in a specific notebook with the original
numbering preserved or collapsed where the original notebooks were doing
nearly the same work.

## Folder layout

```
capstone_viz/
├── README.md                       this file
│
├── prep/                           5 notebooks. Run in order the first time.
│   ├── _config.py                  paths, sample def, name maps, colors
│   ├── 1_data_prep.ipynb           Master.csv → panel.parquet (+ derived cols)
│   ├── 2_clusters_and_correlations.ipynb
│   │                               PCA, cluster aggregation, correlations
│   ├── 3_ml_models.ipynb           LASSO/Ridge/EN/RF + forecast + bootstrap
│   ├── 4_regressions.ipynb         Model 3b + across-samples robustness
│   └── 5_resource_portfolio.ipynb  rent decomposition, portfolios, prod map
│
├── viz/                            4 notebooks, each renders one chart group.
│   ├── _style.py                   shared Plotly style (font, colors, save())

│   ├── viz_2_ml.ipynb              charts 7, 8, 9

│   └── viz_4_robustness.ipynb      charts 18, 19, 21
│
├── artifacts/                      generated by prep, consumed by viz

```

## How to run

### 1. Set the data path

The prep notebooks default to looking for `Master.csv`, `NaturalResource.csv`,
and `clusters1995.csv` in:

```
~/Desktop/GitHub/Capstone/CLEAN/intermediary
```

Change the path either by editing `INTERMEDIARY` in `prep/_config.py`, or
by setting the env var:

```bash
export CAPSTONE_INTERMEDIARY="/your/path/to/intermediary"
```

### 2. Run prep notebooks (one-time, ~5 minutes)

```bash
cd capstone_viz/prep
jupyter nbconvert --to notebook --execute --inplace 1_data_prep.ipynb
jupyter nbconvert --to notebook --execute --inplace 2_clusters_and_correlations.ipynb
jupyter nbconvert --to notebook --execute --inplace 3_ml_models.ipynb
jupyter nbconvert --to notebook --execute --inplace 4_regressions.ipynb
jupyter nbconvert --to notebook --execute --inplace 5_resource_portfolio.ipynb
```

Or just open each in Jupyter and run-all.

To skip the slow ML refit on later runs:

```bash
SKIP_ML_FIT=1 SKIP_BOOTSTRAP=1 jupyter nbconvert --to notebook --execute --inplace 3_ml_models.ipynb
```

### 3. Run viz notebooks (~10 seconds each)

```bash
cd capstone_viz/viz
jupyter nbconvert --to notebook --execute --inplace viz_1_descriptive.ipynb
jupyter nbconvert --to notebook --execute --inplace viz_2_ml.ipynb
jupyter nbconvert --to notebook --execute --inplace viz_3_regression.ipynb
jupyter nbconvert --to notebook --execute --inplace viz_4_robustness.ipynb
```

HTML files land in `outputs/`. Copy them straight to
`projects/capstone/outputs/` in the portfolio repo.

## What's intentionally not here

- **Models 3a, 3c, 3d** — only 3b is used by the portfolio page (chart 12).
  If you want the appendix forest plot back, add a fit block in
  `prep/4_regressions.ipynb` and a chart in `viz_3_regression.ipynb`.
- **OOS R² and prediction-interval charts (15, 16)** — not on the page.
- **Cluster Sankey (chart 11)** — clusters are static, transitions are not
  data, the chart was removed.
- **Forest robustness ML (chart B2 group)** — not on the page.
- **Charts 22 / 23 / 24 / B1 / B3 / B4 / B5** — appendix-only, not on the
  page.
- **NB10 / NB11 / NB12 standalone scripts** — collapsed into prep
  notebooks 3 and 4.

## What to edit if you want to change something

| Want to | Edit |
|---|---|
| Change sample threshold (1% → 2%) | `prep/_config.py` `NR_THRESHOLD`, then re-run prep 1, 3, 4 |
| Change colours | `viz/_style.py` and the cluster colour dicts at the top of each viz notebook |
| Change which countries are case studies | `viz/viz_3_regression.ipynb` `cases` dict — no prep refit |
| Add a regressor to Model 3b | `prep/_config.py` `REG_VARS` (or `INTERACT`), re-run prep 4 |
| Truncate a chart's bars to top N | inside the relevant viz notebook (`.tail(N)` on the dataframe) |
